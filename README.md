###План тестирования приложения для управления личными встречами

1. Введение

 Цель тестирования: Проверить функциональность, безопасность и удобство использования приложения для управления личными встречами.

 Область применения: Тестирование всех основных функций приложения, включая создание, редактирование, удаление встреч, уведомления и создания отчета о встречах.

2. Объект тестирования

 Консольное приложение.

3. Типы тестирования

 Функциональное тестирование: Проверка всех функций приложения.

 Тестирование пользовательского интерфейса: Оценка удобства использования и дизайна.

 Тестирование безопасности: Проверка корректности работы программы при ввод некорректных данных.

4. Тестовые сценарии

 4.1. Функциональные сценарии

 1. Создание встречи

     • Проверить возможность создания встречи.

 2. Редактирование встречи

     • Изменить детали существующей встречи и проверить обновление.

     • Проверить редактирование встречи с недопустимыми данными (ожидаемое поведение: уведомление пользователя о недопустимых данных).

 3. Удаление встречи

     • Удалить встречу и проверить, что она исчезла из списка.

 4. Уведомления

     • Проверить получение уведомлений о предстоящих встречах.

 5. Отчет о встречах.
     
     • Создать отчет о встречах.


4.2. Нефункциональные сценарии

1. Безопасность

   • Проверить возможность создания встречи на прошедшее время.

   • Проверить возможность создания встречи, если она пересекается с другой.

   • Проверить возможность указания даты начала встречи позже даты завершения встречи.
   
   • Проверить возможность не указывать даты при создании или изменения встречи.
   
   • Проверить возможность не указывать имя, номер телефона участника или названия задачи.
